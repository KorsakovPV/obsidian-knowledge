# Инструкции для работы с этим хранилищем Obsidian

Это хранилище (vault) Obsidian. Все заметки — это `.md`-файлы.
Соблюдай правила ниже, чтобы заметки корректно работали в Obsidian.

## Общие правила для всех заметок

- Связывай заметки через вики-ссылки формата `[[Название заметки]]`,
  а не через обычные markdown-ссылки `[текст](файл.md)`.
- В начало каждой заметки добавляй frontmatter:
  ```
  ---
  created: YYYY-MM-DD
  tags: [тег1, тег2]
  ---
  ```
- Используй `#теги` для категоризации, где это уместно.
- Имена файлов делай человекочитаемыми и совпадающими с заголовком (`# Заголовок`) заметки.
- **Не выполняй git-команды** (`git add` / `commit` / `push` / `pull`).
  Синхронизацию полностью делает плагин Obsidian Git. Только создавай и редактируй файлы.

## Структура хранилища

```
.
├── CLAUDE.md          ← этот файл
├── Inbox/             ← свежие, ещё не разобранные заметки
├── Notes/             ← «дозревшие» заметки
├── Daily/             ← ежедневные заметки (если ведутся)
├── Projects/          ← документация по проектам (см. ниже)
└── Attachments/       ← картинки и вложения
```

По умолчанию новые произвольные заметки клади в `Inbox/`.

## Генерация проектной документации

Документация по проектам из `/home/pavel/PycharmProjects` хранится в этом
хранилище, отдельно от кода.

Правила:

- Документацию складывай в `Projects/<имя-проекта>/`.
- Для каждого проекта создавай заметку `Overview.md` как точку входа:
  краткое описание проекта + ссылки `[[...]]` на остальные заметки этого проекта
  (архитектура, API, модули и т. д.).
- Типичный набор заметок на проект:
  - `Overview.md`     — что это за проект, стек, как запустить, оглавление со ссылками.
  - `Architecture.md` — структура, основные модули, как они связаны.
  - `API.md`          — публичные интерфейсы / эндпоинты / ключевые функции (если применимо).
  Добавляй другие заметки по необходимости, всегда связывая их из `Overview.md`.
- Связывай заметки вики-ссылками `[[Название]]`.
- В начало каждой проектной заметки добавляй frontmatter:
  ```
  ---
  project: <имя-проекта>
  created: YYYY-MM-DD
  tags: [project, ...]
  ---
  ```
- Обязательно проставляй тег `#project` и поле `project:` в frontmatter —
  это позволяет собрать общий список проектов одной заметкой (например, через Dataview).

## Как ты обычно вызываешься для документации

Меня (Claude Code) запускают в папке кода, например:
`cd /home/pavel/PycharmProjects/<проект> && claude`
и просят сгенерировать документацию. В этом случае:

1. Анализируй код в текущей рабочей директории.
2. Результат (`.md`-заметки) пиши в `~/Obsidian/Knowledge/Projects/<имя-проекта>/`.
3. Не модифицируй и не коммить сам код проекта без явной просьбы.
